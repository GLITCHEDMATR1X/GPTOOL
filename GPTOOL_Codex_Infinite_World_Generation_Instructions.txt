PASS 17 — HOTFIX: SAFE IMPORTED ACTOR QUARANTINE
================================================================================

Branch:
    git checkout -b pass17-safe-actor-quarantine

Goal:
    Stop imported model failures from crashing the generated game. Bad models should be
    rejected and logged; generated projects must always fall back to procedural testers.

Version:
    0.6.6-pass16 -> 0.6.7-pass17

Generated template:
    panda3d_playable_simulation_template.v6 -> panda3d_playable_simulation_template.v7

Required generated CLI flags:
    --no-imported-actors
    --actor-proof
    --quarantine-bad-actors

Behavior:
    - If --no-imported-actors is present, ignore human_manifest.json and use procedural bodies.
    - If --actor-proof is present, test-load all candidate base_assets in human_manifest.json and
      write a report before assigning any to player characters.
    - If --quarantine-bad-actors is present, failed actor assets should be logged and skipped.
    - If a model fails Actor(...), try loader.loadModel(...).
    - If both fail, reject the asset and fall back.
    - If bounds cannot be read, do not crash. Use default scale.
    - If animations mismatch, do not crash. Load static or use procedural fallback.
    - Never let a single bad male/female model crash the whole generated project.

Add generated outputs:
    reports/actor_import_proof.json
    reports/rejected_human_assets.json

Proof JSON should include per-character:
    actor_loaded
    actor_source_path
    actor_load_mode: actor | static_model | procedural_fallback
    actor_load_errors: []
    rejected_asset_count

Acceptance:
    - With no human_manifest: generated project runs.
    - With human_manifest but --no-imported-actors: generated project runs.
    - With human_manifest and bad fake path: generated project runs, rejects bad path.
    - With human_manifest and real models: generated project runs or falls back safely.
    - Stress proof writes screenshot and proof JSON.
    - No crash_latest.txt for safe failures.
    - Bad actor list is written when failures occur.

Test commands:

    python -S bridge.py generate-game ./GeneratedActorSafeProbe --profile panda3d --command "make a playable simulation with safe imported actors and actor proof" --force

    python ./GeneratedActorSafeProbe/main.py --screenshot-mode --route-proof --stress-proof --actor-proof --quarantine-bad-actors --screenshot-path GeneratedActorSafeProbe/screenshots/actor_safe.png --proof-path GeneratedActorSafeProbe/reports/actor_safe.json

Emergency user test:

    python ./GeneratedActorSafeProbe/main.py --no-imported-actors

Do not move to infinite world generation until this is stable.

PASS 18 — INFINITE WORLD GENERATION CORE
================================================================================

Branch:
    git checkout -b pass18-infinite-world-core

Goal:
    Replace the current small streaming platform with a true deterministic chunk/sector
    streaming system for generated Panda3D templates.

Version:
    0.6.7-pass17 -> 0.6.8-pass18

Generated template:
    panda3d_playable_simulation_template.v7 -> panda3d_infinite_world_template.v1

Core design:
    - Use deterministic chunk coordinates.
    - Active chunk grid follows controlled player.
    - Chunks generate around player.
    - Chunks unload behind player.
    - Same seed + same chunk coordinates = same world.
    - No duplicate chunks.
    - No uncontrolled node buildup.
    - Object/prop budget per chunk.
    - Region/biome rules decide what each chunk contains.
    - Route proof verifies movement across multiple chunks.

Settings additions:
    settings/game_settings.json:

    "world_streaming": {
      "enabled": true,
      "schema_version": "gptool_world_streaming.v1",
      "seed": 12345,
      "chunk_size": 48,
      "active_radius": 2,
      "preload_radius": 2,
      "unload_radius": 3,
      "max_active_chunks": 49,
      "object_budget_per_chunk": 24,
      "terrain_mode": "procedural_mesh_grid",
      "height_mode": "deterministic_noise",
      "biome_mode": "region_weighted",
      "proof_route_chunks": 8
    }

Implement generated classes or helpers:
    DeterministicNoise
    BiomeResolver
    WorldChunk
    WorldStreamer
    ChunkObjectSpawner
    InfiniteWorldProofRunner

Chunk coordinate logic:
    chunk_x = floor(player_x / chunk_size)
    chunk_y = floor(player_y / chunk_size)

Active set:
    all chunks in radius around current chunk.

Unload:
    chunks with distance > unload_radius.

Proof metrics:
    active_chunk_count
    max_active_chunk_count
    generated_chunk_count
    unloaded_chunk_count
    reused_chunk_count
    duplicate_chunk_count
    object_count
    max_object_count
    player_chunk_start
    player_chunk_end
    visited_chunks
    biome_counts
    no_crash_log

Generated proof command:

    python main.py --screenshot-mode --route-proof --stress-proof --world-stream-proof --screenshot-path screenshots/world_stream.png --proof-path reports/world_stream.json

Proof JSON schema:
    gptool_world_stream_proof.v1

Acceptance:
    - proof route crosses at least 8 unique chunks
    - active chunks stay <= max_active_chunks
    - duplicate_chunk_count == 0
    - generated_chunk_count >= active chunks
    - unloaded_chunk_count > 0 after long route
    - screenshot exists
    - no crash log
    - generated project still runs interactively

PASS 19 — INFINITE TERRAIN ELEVATION + LOD
================================================================================

Only after Pass 18 is stable.

Goal:
    Improve terrain shape/performance without breaking proof.

Options:
    A. Procedural mesh grid per chunk with deterministic heights.
    B. Panda3D GeoMipTerrain experiments for larger heightfield chunks.
    C. Hybrid: simple mesh chunks for gameplay + optional GeoMipTerrain mode.

Important research:
    Panda3D GeoMipTerrain uses heightfields and supports LOD around a focal point.
    But do not force GeoMipTerrain into the generator until it passes proof and does
    not cause update stutter or collision issues.

Add settings:
    "terrain_lod": {
      "enabled": false,
      "mode": "mesh_chunks",
      "near": 40,
      "far": 140,
      "height_scale": 8.0,
      "collision_mode": "sampled_height"
    }

Acceptance:
    - Can toggle back to flat/procedural mesh mode.
    - Terrain heights are deterministic.
    - Player has safe ground sampling.
    - No falling forever.
    - Proof checks min/max sampled height and player z stability.
