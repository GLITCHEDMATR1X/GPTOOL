TASK 2 — GENERATED CRASH DIAGNOSTICS
================================================================================

Edit:
    game_builder/template_generator.py

Goal:
Every generated Panda3D project should write useful logs when it crashes, especially during screenshot/proof mode.

Add generated file outputs:
    logs/crash_latest.txt
    logs/runtime_latest.json
    logs/last_controls_state.json
    logs/last_scene_state.json

Add safe generated helpers:
    _safe_vec3_list(value)
    _write_json_safely(path, payload)
    _runtime_state_snapshot()
    _controls_state_snapshot()
    _scene_state_snapshot()
    _write_runtime_diagnostics(reason)
    _write_crash_log(exc, reason="runtime")

Crash log should include:
    - timestamp
    - reason
    - exception type
    - exception message
    - traceback
    - sys.argv
    - cwd
    - Python version
    - Panda3D version if available
    - settings path
    - screenshot path from env
    - proof path from env
    - route-proof requested true/false
    - stress-proof requested true/false
    - active character ID/name if available
    - character positions if available
    - points if available
    - camera distance/height/yaw if available
    - current controls/keys state if available

Important:
    - Diagnostics helpers must never crash themselves.
    - Use try/except inside every diagnostics writer.
    - Do not hide exceptions during development.
    - In screenshot/proof mode, write crash log and exit nonzero.
    - In normal interactive mode, write crash log and print the path.

Suggested wrapping strategy:
    - Keep the generated app simple.
    - Wrap the main GeneratedGame startup/run block with a small top-level guarded entry.
    - If an exception happens before app exists, still write a basic crash log.
    - If app exists, include app state.

Acceptance:
    - For a normal proof run, no crash log should be written.
    - For a forced synthetic crash, crash_latest.txt should be written.
